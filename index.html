What is this?

A reference for designers and front-end developers when it comes to defining the look of Kolibri. For example, answering questions like what type of button should be used for a function, or the color of a button at a particular location in the page.

What do we hope to achieve by following these guidelines?

  1. Give Kolibri a simple, uncluttered, and consistent look.
  2. Make the UX workflow learnable and discoverable.
  3. Provide an immersive learning environment

General Design Principles

Flat design first. Avoid shadows and depth as much as possible. Only use it when absolutely necessary. For example, we apply shadows to the default dropdown list because the dropdown list is likely to expand and cover other kinds of elements. To ensure usability, we give it a shadow to distinguish it from the elements being covered.

Default elements first. We have a list of default elements that closely follow our design guidelines. Use them whenever possible to keep a consistent look. Most of the time, you can get what you want by combining the default elements.

Content first. Ideally, UI elements should reflect the qualities of the content it contains. Since Kolibri is designed to be content agnostic, until we figure out a way to implement content-conscious UI, we should keep our UI as subtle as possible, and emphasize the content. UI is only there to organize the content.

Mobile first. Take touch input into consideration. When a single element cannot be properly displayed on a smartphone screen, decompose it into multiple elements until every element can fit into a smartphone screen. Avoid horizontal scrolling.
