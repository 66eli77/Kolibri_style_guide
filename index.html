What is this?

A reference for designers and front-end dev when it comes to define the look of Kolibri. For example, what type of button should I use for a function and the color of the button at this particular location of the page.

What do we try to achieve by following these guidelines?

  1. Give Kolibri a simple, uncluttered, and consistent look.
  2. Make the UX workflow learnable and discoverable.
  3. Provide a immersive learning environment

General Design Principles

Flat design first. Avoid shadow and depth as much as possible. Only use it in absolute necessity. For example, we give shadow to the defualt dropdown list because the dropdown list is likely expand and cover other different kinds of element. To ensure usability, we give it a shadow to distinguish itself from the covered elements

Default elements first. We have a list of default elements that closely follow our design guidelines. Use them whenever possible to keep a consistent look. Most of the time, you can get what you want by combining the default elements.

Content first. Ideally, UI elements should reflect the quality of the contents it contains. Since Kolibri is designed to be content agnostic, until we figure out a way to implement content consciouse UI, we should keep our UI as subtle as possible, and emphasize on the content. UI is only there to organize the contents

Mobile first. Take touch input into consideration. When a single element cannot be properly displayed on a smartphone screen, decompose it into multiple elements until every element can fit into a smartphone screen. Avoid horizontal scrolling.
